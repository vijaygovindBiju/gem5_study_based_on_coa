Workload Analysis (test.c)

The benchmark was specifically designed to create pathological memory access patterns to stress-test cache associativity and replacement logic. It targets a 16kB L1 Data Cache with 64-byte lines.

Key Constants

CACHE_SIZE16,384 Bytes
LINE_SIZE64 Bytes
TOTAL_LINES256 Lines
CONFLICT_STRIDE4,096 Integers (16kB)

Stress Phases

  • Phase 1 & 2 (Conflict Stress): Accesses 5 memory locations mapping to the same set in a direct-mapped cache.
  • Phase 5 (Replacement Stress): Iterates through 257 distinct lines, exceeding the 256-line capacity.
  • Phase 6 (Working Set Sweep): Steps through working sets from 16 to 257 lines.

Core Stress Loop (Phase 5)

This loop is designed to trigger constant evictions by accessing TOTAL_LINES + 1 distinct memory addresses in a pattern that exposes the difference between LRU and FIFO.

/* Phase 5: MAIN FULLY-ASSOCIATIVE STRESS (257 lines) */ for (r = 0; r < 800; r++) { for (i = 0; i < TOTAL_LINES; i++) total += lru_array[i * INT_PER_LINE]; total += lru_array[TOTAL_LINES * INT_PER_LINE]; /* force eviction */ for (i = TOTAL_LINES - 1; i >= 0; i--) total += lru_array[i * INT_PER_LINE]; total += lru_array[0]; total += lru_array[TOTAL_LINES * INT_PER_LINE]; }

Core Concepts: Mapping & Policies

Cache Mapping Strategies

Mapping defines how a memory address is located within the cache:

  • Direct Mapped (1-way): Each memory address maps to exactly one specific cache line. High conflict potential.
  • Set-Associative (N-way): Memory maps to a "set" containing N lines. A balance between speed and flexibility.

Replacement Policies

When a cache set is full, the policy decides which line to evict:

  • LRU (Least Recently Used): Evicts the line that hasn't been accessed for the longest time. Best for temporal locality.
  • FIFO (First-In, First-Out): Evicts the "oldest" line regardless of access history. Simple but prone to thrashing.

Detailed Analysis & Insights

Brief Summary of Results

1-Way to 2-Way: The most significant jump, reducing the miss rate by nearly 50% (from 3.43% to 1.79% for FIFO, and 1.29% for LRU).

Policy Impact: LRU consistently outperforms FIFO in all associative configurations, especially at 8-way where LRU reaches 99.81% hit rate versus FIFO's 99.45%.

Optimal Choice: 4-Way LRU offers the best balance, capturing over 99.5% of hits with significantly lower complexity than 8-way.

Performance Impact (CPI & AMAT)

The Average Memory Access Time (AMAT) provides a direct measure of the memory hierarchy's efficiency. Our best configuration (8-way LRU) achieved an AMAT of 2.11 cycles, only 5.5% higher than the theoretical ideal of 2 cycles.

The correlation between AMAT and Cycles Per Instruction (CPI) is nearly linear. The transition from 1-way to 8-way LRU reduced CPI from 5.12 to 3.90—a 24% boost in execution efficiency. The "knee" of the performance curve is clearly visible at 4-way associativity, beyond which hardware complexity increases while performance gains diminish.
